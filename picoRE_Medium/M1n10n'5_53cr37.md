Personal learning outcome: 
- For all RE challenges that given executable file, decompile first. 
>Rather than, [dogbolt](https://dogbolt.org/), use [decompiler.com](https://www.decompiler.com/) is better here, cause the file size is big.

|tools|Use Case|
|Dogbolt|File size less than 2MB|
|decompiler.com|Support file types:
- EXE or DLL,
- JAR or CLASS,
- APK,
- XAPK or DEX,
- PYC or PYO,
- LUAC or LUB,
- SMX or AMXX |

- For APK file, look for ```/resources/res/values``` which store files that define values used by an app
>Eg: colour.xml, strings.xml(text display in UI), dimes.xml(margin, padding), styles.xml, array.xml(array of str, int),...
- In CTF case, strings.xml will be focus to find flag.

- When too many file in dir, can use ```grep -rni VALUE``` to find the file/dir that wanna to go next.
>-r: Recursively searches through directories.
>-n: Displays the line number of each match.
>-i: Makes the search case-insensitive.
